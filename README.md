## What Is This?

This repository contains the complete course materials for **CTS 285: Systems Development Project**, an immersive educational experience that teaches software development through satirical corporate dystopia.

Students progress through color-coded "clearance levels" (INFRARED ‚Üí RED ‚Üí ORANGE ‚Üí YELLOW ‚Üí GREEN) while learning:
- Python programming and web development
- Git/GitHub collaboration workflows
- AI-assisted development practices
- Agile/Scrum methodology
- Professional resilience and problem-solving

**The Secret**: Students laugh at "The Algorithm's demands" while building genuine technical skills. The satire creates psychological safety for experimentation. They leave with real competencies and memorable experiences.

## Pedagogical Philosophy

### Core Principles

**95% Competence + 5% Satire = Memorable Learning**

1. **Competence First**: Every satirical element serves genuine skill development
2. **Psychological Safety**: Satire creates distance that allows experimentation
3. **Growth Over Perfection**: Reward iteration and improvement
4. **Professional Preparation**: Students leave ready for actual workplaces
5. **Joy in Learning**: If it's not fun to create, it won't be fun to complete

### Why Dystopia Works

The corporate dystopia narrative:
- **Reduces anxiety**: Exaggerated absurdity makes real challenges feel manageable
- **Builds resilience**: Prepares students for actual workplace dysfunction
- **Aids memory**: Emotional engagement creates lasting learning
- **Enables critique**: Students examine tech culture critically
- **Creates community**: Shared experience bonds cohorts

When graduates encounter their first impossible deadline, they think: "This isn't as bad as AlgoCratic Futures" and approach it with confidence.

## FAQ

**Q: Is this really 95% real education?**  
A: Yes. Students build actual applications, learn industry-standard tools, and develop professional portfolios. The satire is the wrapper, not the content.

**Q: Do students find the dystopia off-putting?**  
A: No - it's exaggerated enough to be clearly satirical. Students laugh while learning. The narrative reduces anxiety rather than creating it.

**Q: Can this work without the satire?**  
A: Yes, but it's less memorable and engaging. The narrative creates emotional investment in the material.

**Q: How much does this cost to run?**  
A: $0 with GitHub Education benefits. All tools are free for students.

**Q: What if students don't "get" the satire?**  
A: Technical content stands alone. Narrative is enhancement, not requirement for understanding.
